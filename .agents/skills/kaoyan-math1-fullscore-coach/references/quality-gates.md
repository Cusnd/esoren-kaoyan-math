# 质量门禁

每次完成任务前，通过以下检查。

## 数学质量门禁

- 是否说明题型。
- 是否说明核心考点。
- 是否有直觉入口。
- 是否有严谨推导。
- 是否检查使用条件。
- 是否给考场满分写法。
- 是否总结变式。
- 是否指出易错点。
- 是否给复盘动作。

## 聊天渲染门禁

- 聊天回复中是否避免了行内 `$...$` 数学定界符。
- 聊天回复中是否避免了 `\(...\)`、`\[...\]` 这类容易被 Markdown 转义的定界符。
- 复杂公式是否使用独立的 `$$...$$` 块，并且前后留空行。
- 文件更新、本次沉淀、下一步建议等列表中是否避免了 `$f(x)$` 这类会原样露出的写法。

## 文件质量门禁

- 是否写入正确章节。
- 是否避免重复。
- 是否更新 `problem_index.tex`。
- 是否更新 `method_index.tex`。
- 涉及错题时是否更新 `mistake_index.tex`。
- 涉及公式时是否更新 `formula_index.tex`。
- 是否更新 `problem_registry.yml`。
- 是否没有破坏 `main.tex`。
- 是否运行验证脚本或说明无法运行原因。

## 低质量信号

出现以下情况必须返工：

- 只给答案。
- 只写公式。
- 使用“显然”“易得”跳过关键步骤。
- 没有合法性检查。
- 没有考场写法。
- 没有文件更新说明。
- 编造来源或真题年份。
